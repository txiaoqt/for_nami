# 3.4 Test Methodology and Procedures

Testing verifies that LYDO Connect behaves correctly across its current organization-focused workflow.

## 3.4.1 Unit Testing

Unit testing checks isolated functions and components such as:
- authentication and policy gate logic
- profile field validation
- compliance document upload helpers and preview behavior
- budget request form validation
- liquidation visibility checks
- YPOP scoring, status, and submission helpers
- news release preview helpers
- template/file utilities
- notification formatting and status helpers

## 3.4.2 Integration Testing

Integration testing checks that connected modules work together:
- user account and organization profile records
- policy acceptance and protected-route access
- document submissions and stored files
- budget request CRUD and attached file uploads
- approved budgets and liquidation record visibility
- YPOP entries, proof files, and admin validation updates
- YORP-related organization registry field updates
- barangay budget allocation summary loading and filter behavior
- admin news release CRUD and preview routing
- template management and compliance submission linkage
- notifications and activity log updates

## 3.4.3 System Testing

System testing evaluates the application as a whole using realistic user flows:
- sign in and accept the active policy
- complete and save an organization profile
- submit required documents
- create, edit, delete, save, and submit a budget request
- upload liquidation files after budget approval
- create, update, submit, and track a YPOP entry
- review budget monitoring results by district and barangay
- open news release preview pages
- manage templates, YORP-related organization fields, and admin review screens

## 3.4.4 Acceptance Testing

Acceptance testing determines whether the final build matches the expected organization-user and admin workflow and is understandable to target users and reviewers.

## 3.4.5 Performance Testing

Performance testing checks page responsiveness, file preview loading, list rendering, filtering behavior, and record retrieval under normal usage.

## 3.4.6 Security Testing

Security testing verifies authenticated access, role-based route protection, policy gating, safe handling of organization records, and admin-only access to monitoring, validation, template, and registry actions.

## 3.4.7 Usability Testing

Usability testing checks whether organization users and admins can complete the actual workflow on mobile and desktop without confusion.

## 3.4.8 Compatibility Testing

Compatibility testing verifies that the app works on current desktop and mobile browsers with responsive layouts.
